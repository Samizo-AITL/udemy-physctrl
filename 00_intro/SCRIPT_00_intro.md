## Core Message 1 — Control is a Design Activity (1:00–1:40)

ここでの制御は、
パラメータ調整の作業ではありません。

制御は、
「どういう構造にするか」を決める設計行為です。

まずは、シンプルな PID を動かします。

## Core Message 2 — Physical Models Define Constraints (1:40–2:20)

次に、物理モデルを見ます。

物理モデルは、
制御を難しくするためにあるのではありません。

「なぜ同じ制御が通用しないのか」
その理由を教えてくれます。

## Core Message 3 — FSM Is a Supervisory Layer (2:20–3:00)

そして FSM です。

FSM は、制御そのものを計算しません。
どの制御を使うかを選びます。

状態によって、やるべきことが変わる。
それを扱うのが FSM です。
